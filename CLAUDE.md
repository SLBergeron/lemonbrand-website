# Lemonbrand Website - Project Context

## Overview

This is the marketing website for Lemonbrand, an AI-powered automation agency with the tagline "Automate your way to freedom." Core thesis: **businesses don't need to rent software anymore—the tools they need can be built fast and owned forever.**

## Tech Stack

- Next.js 14+ (App Router)
- TypeScript
- Tailwind CSS
- Convex (database)
- Better Auth (authentication)
- Resend (email)
- Stripe (payments)
- Cal.com (scheduling)

## Project Structure

```
app/
  (marketing)/     # Public marketing pages
  api/             # API routes
  clients/         # Client-specific pages
  dashboard/       # User dashboard
constants/         # Shared constants and data
components/        # Reusable components
lib/               # Utilities, auth, payments
```

## Frameworks & Methodologies

### The Build Stack (3-Part Request Structure)

1. **Context** — What exists, what you're trying to do, your constraints
2. **Task** — The specific thing you want built
3. **Constraints** — How you want it done

### The 2-3 Exchange Pattern

The iterative cycle that closes the gap between "what I got" and "what I wanted." Key insight: **Starting over is almost never the answer.**

### The Communication Framework

> "Two people can give Claude the exact same request and get completely different outputs. The difference isn't the prompt. It's the context."

The skill isn't coding—it's communication.

## ICP (Ideal Customer Profile)

### Primary: The Operator-Builder

**Demographics:**
- Small business owner, 2-50 employees
- Revenue: $500K-$5M annually
- Industry: Service-based (agencies, contractors, consultants, gyms, property management)
- Location: North America (Ontario compliance angle for Canadian market)

**Psychographics:**
- Frustrated by SaaS sprawl ($500-$2K/month on software subscriptions)
- Values ownership and control over convenience
- Practical, not theoretical—wants results, not features
- Understands their business deeply but thinks they "can't code"
- Time-constrained, willing to pay for speed

**Pain Points:**
- Software that has 80% of needed features, with the 20% gap causing friction
- Vendor lock-in anxiety
- Escalating subscription costs
- Tools that don't integrate with each other
- Waiting months for custom development from traditional agencies

**Trigger Events:**
- Annual software renewal shock ($X00 → $X,000)
- Scaling past what current tools can handle
- New compliance requirements (e.g., O. Reg. 476/24)
- Competitor has better systems/automation

**Jobs to Be Done:**
1. Automate repetitive workflows that eat 5-20 hours/week
2. Create unified dashboards instead of checking 8 tools
3. Build client-facing portals that look professional
4. Replace manual processes before hiring another person

### Secondary: The Aspiring Builder

**Demographics:**
- Solo consultant, freelancer, or side-project operator
- Revenue: $50K-$300K
- Technical curiosity but no formal coding background
- Age: 30-50

**Psychographics:**
- Wants to build their own tools without hiring developers
- Values learning and self-sufficiency
- Active in online communities (Twitter/X, newsletters, Discord)
- Following "build in public" creators

**Entry Point:** 7-Day Sprint ($297) → 8-Week Program → Builders Club retention

### Tertiary: The Agency Owner

**Demographics:**
- Marketing, creative, or consulting agency
- 5-20 person team
- Revenue: $1M-$10M
- Multiple clients with similar needs

**Psychographics:**
- Sees custom tools as competitive advantage
- Wants to white-label or resell tools to clients
- Values efficiency gains that compound across clients

**Products for This ICP:**
- Custom builds that become productized
- Client Portal (coming soon)
- Template licensing opportunities
